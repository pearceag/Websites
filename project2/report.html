1. This service is not a to-do list. It is a service similar to how playlists store information.

2. My app uses MySQL and only uses one table called playlist-table.

3. My app provides an Express-based web service for interacting with the database.

4. My web service is available via https://project2.lexiepearce.me:8443/.

5. The endpoints of the web service and any parameters are named approriately.

6. My web service sends and receives complex data as JSON.

7. The full spectruc of CRUD operations are supported.

8. I use ufw to block all ports, but the ones I need for SSH and my allowed web servers.

9. I created a Nginx server that allows global, encrypted access to my web service.

10. My web service supports cross-orgin resource sharing using custom middleware and an options wildcard endpoint.

11. My web service is started using the process manager pm2.

12. My service is stored in my github repo: https://github.com/pearceag/cs347/tree/master/project2.

13. My database's creationg and schema scripts are included in my repository.

14. My github repo shows a stead flow of commits and pushes.

15. My service responds to https://project2.lexiepearce.me:8443/report.html.
